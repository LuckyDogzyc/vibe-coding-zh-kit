# Prime：加载项目上下文

## 目标

通过分析项目结构、文档和关键文件，建立完整理解。

## 流程

### 1. 分析项目结构

列出所有受版本控制文件：
!`git ls-files`

展示目录结构：
在 Linux 上运行：`tree -L 3 -I 'node_modules|__pycache__|.git|dist|build'`

### 2. 阅读核心文档

- 读取 PRD.md 或同类规格文档
- 读取 CLAUDE.md 或同类全局规则
- 读取项目根目录与关键子目录 README
- 读取架构文档（如有）
- 读取 drizzle 配置以理解数据库结构

### 3. 识别关键文件

基于结构识别并阅读：
- 主入口文件（main.py、index.ts、app.py 等）
- 核心配置（pyproject.toml、package.json、tsconfig.json）
- 关键模型/schema 定义
- 关键服务或控制器文件

### 4. 理解当前状态

检查近期活动：
!`git log -10 --oneline`

检查当前分支与状态：
!`git status`

## 输出报告

请给出简洁总结：

### 项目概览
- 应用目的与类型
- 主要技术与框架
- 当前版本/状态

### 架构
- 总体结构与组织方式
- 关键架构模式
- 重要目录用途

### 技术栈
- 语言与版本
- 框架与主要库
- 构建工具与包管理器
- 测试框架

### 核心原则
- 观察到的代码风格和约定
- 文档标准
- 测试策略

### 当前状态
- 当前分支
- 近期变更重点
- 即时风险或注意事项

**请使用清晰标题和要点，确保可快速扫描。**

## 输出后必须附加

在回复末尾给出「下一步可选流程」（2-3 项），示例：
1. create-rules（生成规则）
2. create-prd（生成需求文档）
3. plan-feature（输出实施计划）
