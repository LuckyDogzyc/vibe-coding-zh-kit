# Execute：按计划实现

## 要执行的计划

读取计划文件：`$ARGUMENTS`

## 执行说明

### 1. 阅读并理解

- 完整阅读计划，不要跳读
- 理解全部任务及其依赖关系
- 记下需要执行的验证命令
- 审阅测试策略

### 2. 按顺序执行任务

对于“Step by Step Tasks”中的每一个任务：

#### a. 定位任务
- 确认目标文件与要执行的动作
- 若是修改，先读取相关现有文件

#### b. 实现任务
- 严格按照计划中的细节实现
- 与现有代码模式保持一致
- 按需要补充类型提示和文档
- 按需要加入结构化日志

#### c. 边做边验证
- 每改动一个文件后先检查语法
- 确保 import 正确
- 验证类型定义正确

### 3. 实施测试策略

完成实现任务后：

- 创建计划中要求的全部测试文件
- 实现计划中要求的全部测试用例
- 按计划采用测试方法
- 确保测试覆盖边界情况

### 4. 运行验证命令

按计划中的顺序执行全部验证命令：

```bash
# 按计划逐条执行命令
```

如果有命令失败：
- 修复问题
- 重新运行该命令
- 通过后再继续

### 5. 最终核对

完成前确认：

- ✅ 计划中的所有任务已完成
- ✅ 所有测试已创建并通过
- ✅ 所有验证命令通过
- ✅ 代码符合项目约定
- ✅ 必要文档已更新
