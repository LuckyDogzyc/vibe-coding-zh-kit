# Execute: 按计划实施

## 要执行的计划

读取计划文件：`$ARGUMENTS`

## 执行说明

### 1）完整阅读与理解

- 完整阅读计划文件，不要只看摘要
- 理解所有任务及其依赖关系
- 记录计划中的验证命令
- 理解计划的测试策略与通过标准

### 2）按顺序执行任务

对“Step by Step Tasks”中的每个任务逐一执行：

#### a. 定位任务
- 确认目标文件与操作类型（CREATE / UPDATE / ADD / REMOVE / REFACTOR / MIRROR）
- 若为修改，先读取相关现有文件

#### b. 实现任务
- 严格遵循计划中的实现细节
- 保持与现有代码风格、目录约定、命名约定一致
- 需要时补充类型定义、注释和结构化日志

#### c. 过程校验
- 每次改动后先做快速校验（语法、导入、类型）
- 发现问题立即修复，不要累积到最后

### 3）实施测试策略

当实现任务完成后：

- 创建计划中要求的全部测试文件
- 实现计划中要求的全部测试用例
- 覆盖关键路径与边界条件
- 确保测试结构符合项目现有模式

### 4）执行验证命令

按计划列出的顺序执行全部验证命令。

```bash
# 按计划逐条执行
```

如果任意命令失败：
1. 修复问题
2. 重跑失败命令
3. 全部通过后再继续

### 5）最终检查

完成前必须确认：

- ✅ 计划中的任务全部完成
- ✅ 测试创建完毕且通过
- ✅ 验证命令全部通过
- ✅ 代码符合项目约定
- ✅ 相关文档已更新（如需要）

## 注意

- 如果遇到计划未覆盖的问题，请记录并说明处理方式
- 若必须偏离计划，请明确写出偏离原因
- 不要跳过验证步骤
