# Create Global Rules

目标：分析项目并生成项目级 `CLAUDE.md` 规则文件。

## 阶段 1：识别（DISCOVER）

- 判断项目类型（全栈/前端/API/库/CLI/Monorepo/脚本）
- 读取关键配置（package.json / tsconfig / *.config.*）
- 梳理目录结构与边界（源码、测试、共享模块、配置）

## 阶段 2：分析（ANALYZE）

- 技术栈（运行时、框架、数据库、测试、构建、Lint）
- 代码模式（命名、结构、错误处理、类型定义、测试风格）
- 关键文件（入口、核心逻辑、共享工具、类型）

## 阶段 3：生成（GENERATE）

输出 `CLAUDE.md`（项目根目录），包含：
1. 项目概览
2. 技术栈
3. 常用命令（开发/构建/测试/校验）
4. 目录结构
5. 代码约定
6. 关键文件

可选：架构说明、接口说明、组件规范、数据库模式、按需参考文档。

## 阶段 4：结果汇报

输出：
- 已生成文件：`CLAUDE.md`
- 识别到的项目类型
- 技术栈摘要
- 结构摘要
- 后续建议（补充项目特定规则、删减不适用章节等）
