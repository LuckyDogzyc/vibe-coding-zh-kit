# Create Global Rules

通过分析代码库提炼模式，生成 `CLAUDE.md`。

## 目标

输出项目级全局规则，涵盖：
- 项目是什么
- 技术栈
- 代码组织方式
- 约定与模式
- 构建、测试、验证方式

## Phase 1: DISCOVER

- 识别项目类型
- 分析配置文件
- 映射目录结构

## Phase 2: ANALYZE

- 提炼技术栈（运行时、框架、数据库、测试、构建、lint）
- 提炼代码模式（命名、结构、错误、类型、测试）
- 识别关键文件（入口、配置、核心逻辑、共享工具）

## Phase 3: GENERATE

- 以 `.agents/CLAUDE-template.md` 为起点（若存在）
- 输出到项目根目录：`CLAUDE.md`
- 只保留与项目相关且高价值内容，避免冗长

建议章节：
1. 项目概览
2. 技术栈
3. 常用命令
4. 目录结构
5. 代码模式
6. 关键文件

## Phase 4: OUTPUT

输出汇总：
- 生成文件路径
- 项目类型
- 技术栈摘要
- 结构摘要
- 下一步建议

## 输出后必须附加

在回复末尾给出「下一步可选流程」（2-3 项），示例：
1. create-prd（生成需求文档）
2. plan-feature（输出实施计划）
3. init-project（初始化环境）

## Tips

- 保持简洁可扫描
- 不重复已有文档，优先链接
- 聚焦“执行约定”，不是百科
- 随项目演进持续更新
