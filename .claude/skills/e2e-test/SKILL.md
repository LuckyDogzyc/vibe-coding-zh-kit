# E2E 全链路测试流程

## 0. 预检查

1. 平台检查：仅 Linux / WSL / macOS
2. 前端可访问性检查：必须存在浏览器可访问 UI
3. `agent-browser` 安装与可用性检查

若任一不满足，停止并明确报错原因。

## 1. 并行情报收集（建议 3 路）

- 子任务 A：应用结构与用户旅程（启动方式、页面、交互路径）
- 子任务 B：数据库结构与数据流（写入/更新/删除验证点）
- 子任务 C：潜在 Bug 扫描（逻辑、UI/UX、数据一致性、安全）

## 2. 启动应用

- 按调研结果安装依赖并启动开发服务（后台运行）
- `agent-browser open <url>` 确认页面可达
- 首屏截图留存：`e2e-screenshots/00-initial-load.png`

## 3. 建立任务清单

按“每条用户旅程”创建任务，包含：
- 步骤
- 预期结果
- 数据库验证 SQL/脚本
- 关联风险点

附加任务：多视口响应式测试。

## 4. 执行测试

### 4.1 浏览器侧

对每一步交互都执行：
1. snapshot
2. 交互
3. wait
4. 截图（按旅程分目录）
5. 检查 console/errors

### 4.2 数据库侧

凡是会改数据的操作都要验证：
- 是否正确创建/更新/删除
- 字段值是否匹配输入
- 关系是否完整
- 是否产生脏数据/重复数据

### 4.3 发现问题即闭环

- 记录现象与证据
- 修复代码
- 重测失败步骤
- 补截图确认修复

### 4.4 响应式

至少覆盖：
- Mobile: 375x812
- Tablet: 768x1024
- Desktop: 1440x900

## 5. 收尾

- 停止开发服务
- `agent-browser close`

## 6. 汇报

输出简明统计：
- 测试旅程数
- 截图数
- 问题数（已修复 / 未修复）

并可选导出 `e2e-test-report.md`（逐旅程明细 + DB 校验 + 证据路径）。
